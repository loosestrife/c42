How to use

  • copy c42.json, options.h to your project and npm install c42
  • npx c42trans --options c42.json --options-h options.h file.c -o file-transpiled.c
  • extend the transpiler to support your favorite construct, try to do it by injecting --extensions my-extensions.js, send me a pr so everyone can use it!
  • maybe rename your files .c42, then extend the clangd language server to read c42.json to know how to parse your projects .c42 files

The ISO C committee is more likely to standardize features that are widely used, so make sure your feature is widely used!

Why?

  • It is largely a historical accident that relational algebra led to SQL and subsequently every query language has been mostly SQL
  • In the early 90's, before everyone had a network of a cell phone and a laptop which were recursively a network of cores, the academics came up with a "state transitional algebra" which was then implemented in haskell, c#, vala, nim, go, vlang, rust, odin, zig, swift,
  • Function coloring is inevitable, but feel free to make your own state machine with your own runtime and function coloring like youre already doing, and send me a patch to glue your system to all the others.
  • C needs a babel stage between everyone knowing what the syntax should be and the winning version being ISO C42. That babel stage is C42.
